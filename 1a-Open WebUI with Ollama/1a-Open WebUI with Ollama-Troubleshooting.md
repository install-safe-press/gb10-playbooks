# 常見問題排除指南

---

## 透過 NVIDIA Sync 設定時的常見問題

| 症狀 | 原因 | 解決方式 |
|------|------|----------|
| `docker ps` 指令顯示「權限被拒絕」 | 使用者不在 `docker` 群組中 | 完整執行 Step 1 的所有指令，包括重啟終端機 |
| 瀏覽器沒有自動開啟 | 自動開啟設定未啟用 | 手動在瀏覽器輸入 `http://localhost:12000` |
| 模型下載失敗 | 網路連線問題 | 確認網路連線正常後，重新嘗試下載 |
| 容器內偵測不到 GPU | 啟動指令缺少 `--gpus=all` 參數 | 使用正確的啟動腳本重新建立容器 |
| Port 12000 已被佔用 | 另一個應用程式正在使用該埠位 | 在應用程式設定中更換埠位，或先停止衝突的服務 |

---

## 手動設定時的常見問題

| 症狀 | 原因 | 解決方式 |
|------|------|----------|
| `docker ps` 指令顯示「權限被拒絕」 | 使用者不在 `docker` 群組中 | 完整執行 Step 1，包括登出並重新登入；或在指令前加上 `sudo` |
| 模型下載失敗 | 網路連線問題 | 確認網路連線正常後，重新嘗試下載 |
| 容器內偵測不到 GPU | 啟動指令缺少 `--gpus=all` 參數 | 使用正確的指令重新建立容器 |
| Port 8080 已被佔用 | 另一個應用程式正在使用該埠位 | 修改 Docker 指令中的埠位設定，或先停止衝突的服務 |

## 📝 補充說明：DGX Spark 記憶體架構

DGX Spark 採用**統一記憶體架構（UMA, Unified Memory Architecture）**，支援 GPU 與 CPU 之間**動態共享記憶體**。

由於許多應用程式仍在陸續更新以完整支援 UMA，即使在 DGX Spark 的記憶體容量範圍內，仍可能偶爾遇到記憶體相關問題。

**若發生記憶體問題，可手動清除緩衝區快取：**

```bash
sudo sh -c 'sync; echo 3 > /proc/sys/vm/drop_caches'
```

> ⚠️ 此指令會釋放系統的頁面快取、dentries 與 inode 快取，執行後系統下一次讀取資料會稍慢（需重新載入快取），屬正常現象。
